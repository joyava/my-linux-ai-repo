  • Copy ID code from google keep

How to install appimage for Ubuntu?

LM Studio is another application that allows you to chat offline.

  • Search for LM Studio on your browser.
  • Download LM_Studio-or-whatever.AppImage 

    For Linux Mint or Ubuntu 22.04
    sudo apt install libfuse2
    chmod +x LM_Studio-or-whatever.AppImage
    ./LM_Studio-or-whatever.AppImage
    For Ubuntu 24.04
    https://itsfoss.com/cant-run-appimage-ubuntu/
    sudo apt install libfuse2t64
    Ubuntu 24.04 (maybe even earlier version) inhibits you from installing file.appimage
    https://github.com/electron/electron/issues/42510
    You need to deactivate appimage restriction imposed by Ubuntu
    sudo sysctl -w kernel.apparmor_restrict_unprivileged_userns=0
    Now you can install and run appimage files
    chmod +x LM_Studio-or-whatever.AppImage
    ./LM_Studio-or-whatever.AppImage

How to fix date & time problem in Linux Mint

I got help from ChatGPT 


    Check and sync with NTC server
    sudo apt install ntpdate
    sudo ntpdate pool.ntp.org
    install Fake-hwclock from the appstore
    If all fail, manually run command line
    sudo date --set="Thu Jan 12 11:30:46"
    Because sudo require password, I asked ChatGPT:
    How to run sudo command "sudo ntpdate pool.ntp.org" without being prompted for a password? 
    You can configure the sudoers file to allow specific commands to be executed without authentication. 
    Here's how you can do it:
    sudo visudo
    your_username ALL=(ALL) NOPASSWD: /usr/sbin/ntpdate
    Save and Exit
    Run the command 
    sudo /usr/sbin/ntpdate pool.ntp.org
    I put the command in a script
    #!/bin/bash
    sudo /usr/sbin/ntpdate pool.ntp.org
    Finally, it works. 
    I do not need to run daily sudo date --set="Thu Jan 12 11:30:46"

Ubuntu, Close Lid, Screen Off

Q: Can the 'lid close' switch be configured somehow to simply turn off the screen but otherwise the laptop is running as normal? 


        Step 1:

        Edit the grub config sudo nano /etc/default/grub. Change the line
        I've got it working by adding consoleblank=5 to the kernel options and changing the HandleLidSwitch and HandleLidSwitchExternalPower to ignore in the /etc/systemd/logind.conf.

        Edit the grub config sudo nano /etc/default/grub. Change the line
        GRUB_CMDLINE_LINUX_DEFAULT="quiet consoleblank=5"

        This turns the screen off after 5 seconds. When you hit the keyboard, it will turn on again.
        Run sudo update-grub to activate your changes.

        Next:
        Edit the sudo nano /etc/systemd/logind.conf and change HandleLidSwitch and HandleLidSwitchExternalPower to ignore and remove the #.
        HandleLidSwitch=ignore HandleLidSwitchExternalPower=ignore
        Now reboot

What to do if you see this error on steam deck?

Scary pop up message upon booting Steam Deck 


        x x x x x x x x x x x x  x x x x x x x x x x x  x x x x x x x x 
        Problem: I Bricked my Steam Deck because of Dual Boot installation
        
        Upon reboot, this SCARY message pop up: "fTPM data error. 
        Select [No] to keep fTPM data and continue boot or
        Select [Yes] to reset fTPM. 
        If BitLocker or encryption is enabled, the system will not boot without a recovery key.
        Yes or No"
        
        What to do? don't click YES because you will brick your Steam Deck.
        By default Windows 11 automatically sets Bitlocker and encryption enable. 
        My answer: boot to bios. In Menu/Advance/Peripheral, I disable fTPM (firmware TPM). 
        When I rebooted, it did not show that scary error message again. 

        Update: imo, if I replace the current ssd with the original ssd,
        I should get back to the original state with no scary message pop up!
        So, keep your original SSD in a safe place just in case you need it
