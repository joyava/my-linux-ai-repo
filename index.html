Install Syncthing

Dead Link

I want to keep a backup copy of my work in another computer. This is where syncthing come in. sudo apt install syncthing sudo systemctl enable syncthing@joyava.service sudo systemctl start syncthing@joyava.service sudo systemctl status syncthing@joyava.service On your browser type http://localhost:8384 Click setting, click GUI tab, Enter your username and password. Click save. Repeat these steps on another ubuntu laptop2 (I have Ubuntu on a Raspberry Pi5) Get ID from Ubuntu laptop1 and Enter that ID into Ubuntu laptop2 How do you do that? Click action, click show ID, copy to google keep this ID On Ubuntu laptop2: Copy ID code from google keep click "add remote device" at the bottom of the page You paste ID code into this pop up window. click save. On Ubuntu laptop1: click "add device" on pop up message change device name to "laptop2". click save. Now you sit back and watch the "remote device" box When it shows "connected", you are done type ls on your terminal you should see a Sync folder along with Downloads and Pictures folder The default Sync folder is at ~/Sync The "default folder" entry on your browser shows /home/username/Sync click "edit" button, it popup a windows with 3 tabs click sharing tab, then tick the box click file_version tab, choose "simple file version. click save. On Ubuntu laptop2: click share button

How to install appimage for Ubuntu?

Dead Link

LM Studio is another application that allows you to chat offline. Search for LM Studio on your browser. Download LM_Studio-or-whatever.AppImage For Linux Mint or Ubuntu 22.04 sudo apt install libfuse2 chmod +x LM_Studio-or-whatever.AppImage ./LM_Studio-or-whatever.AppImage For Ubuntu 24.04 https://itsfoss.com/cant-run-appimage-ubuntu/ sudo apt install libfuse2t64 Ubuntu 24.04 (maybe even earlier version) inhibits you from installing file.appimage https://github.com/electron/electron/issues/42510 You need to deactivate appimage restriction imposed by Ubuntu sudo sysctl -w kernel.apparmor_restrict_unprivileged_userns=0 Now you can install and run appimage files chmod +x LM_Studio-or-whatever.AppImage ./LM_Studio-or-whatever.AppImage
